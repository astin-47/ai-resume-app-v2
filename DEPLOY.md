# 🚀 Deployment Guide — AI Resume Builder

This guide gets your app live on the internet using:
- **Render** — hosts the Python backend (FastAPI)
- **Vercel** — hosts the frontend (index.html)

Both have free tiers. Total cost to deploy: **$0**.

---

## 📁 Your Final File Structure

Before deploying, make sure your GitHub repo looks like this:

```
ai-resume-app/
├── main.py              ← FastAPI backend
├── index.html           ← Frontend
├── build.js             ← Vercel build script (injects API URL)
├── vercel.json          ← Vercel config
├── render.yaml          ← Render config
├── requirements.txt     ← Python packages
├── .env                 ← Local secrets (NEVER commit this)
├── .env.example         ← Safe template (commit this)
└── .gitignore           ← Must include .env
```

## PART 1 — Push Your Code to GitHub

### Step 1.1 — Make sure .gitignore is correct

Your `.gitignore` must contain `.env` so your API key is never uploaded:

```
.env
__pycache__/
*.pyc
venv/
.venv/
api-config.js
```

> **Important:** Also add `api-config.js` to `.gitignore` — this file is
> auto-generated by Vercel and should not be committed.

### Step 1.2 — Commit and push all files

```bash
git add .
git commit -m "feat: add deployment config for Render and Vercel"
git push origin main
```

## PART 2 — Deploy the Backend on Render

Render will host your FastAPI Python server.

### Step 2.1 — Create a new Web Service

1. Go to [render.com](https://render.com) and log in
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub account if you haven't already
4. Select your **ai-resume-app** repository
5. Click **"Connect"**

### Step 2.2 — Configure the service

Render will auto-detect your `render.yaml` and fill most fields.
Double-check these settings:

| Setting | Value |
|---|---|
| **Name** | ai-resume-backend |
| **Region** | Oregon (US West) |
| **Runtime** | Python 3 |
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `uvicorn main:app --host 0.0.0.0 --port $PORT --workers 2` |
| **Plan** | Free |

> ⚠️ **LibreOffice on Render:**
> The free plan does **not** allow custom apt-get installs in render.yaml.
> PDF export will fall back to returning a DOCX file on the free tier.
> To enable PDF on Render, upgrade to the Starter plan ($7/month) and
> the `apt-get install -y libreoffice` build command will work.

### Step 2.3 — Add environment variables

In the Render dashboard, go to your service → **Environment** tab → **Add Environment Variable**:

| Key | Value |
|---|---|
| `OPENAI_API_KEY` | `sk-your-real-key-here` |
| `ALLOWED_ORIGINS` | Leave blank for now — you'll fill this after Vercel deployment |

### Step 2.4 — Deploy

Click **"Create Web Service"**. Render will:
1. Clone your GitHub repo
2. Install Python packages
3. Start the server

This takes about **3–5 minutes** the first time.

### Step 2.5 — Copy your Render URL

When deployment finishes, you'll see a URL like:
```
https://ai-resume-backend.onrender.com
```

**Copy this URL — you need it for Vercel.**

### Step 2.6 — Test your backend

Open your browser and visit:
```
https://ai-resume-backend.onrender.com
```

You should see:
```json
{"status": "ok", "message": "AI Resume Builder v3 is running ✅"}
```

If you see that — your backend is live! 🎉

## PART 3 — Deploy the Frontend on Vercel

Vercel will host your `index.html` and make it available at a public URL.

### Step 3.1 — Import your project

1. Go to [vercel.com](https://vercel.com) and log in
2. Click **"Add New..."** → **"Project"**
3. Find and select your **ai-resume-app** repository
4. Click **"Import"**

### Step 3.2 — Configure the project

Vercel auto-detects `vercel.json`. Set these:

| Setting | Value |
|---|---|
| **Framework Preset** | Other |
| **Root Directory** | `.` (leave as default) |
| **Build Command** | `node build.js` |
| **Output Directory** | `.` |

### Step 3.3 — Add environment variable

Still on the configure screen, click **"Environment Variables"** and add:

| Key | Value |
|---|---|
| `RENDER_API_URL` | `https://ai-resume-backend.onrender.com` |

Replace the URL with your actual Render URL from Step 2.5.

### Step 3.4 — Deploy

Click **"Deploy"**. Vercel will:
1. Clone your repo
2. Run `node build.js` (creates `api-config.js` with your Render URL)
3. Serve `index.html` at a public URL

This takes about **30–60 seconds**.

### Step 3.5 — Copy your Vercel URL

You'll get a URL like:
```
https://ai-resume-forge.vercel.app
```

**Open it in your browser — your app is live!** 🎉

---

## PART 4 — Lock Down CORS (Security Step)

Now that you have both URLs, update Render to only accept requests from your Vercel domain.

### Step 4.1 — Update ALLOWED_ORIGINS on Render

1. Go to Render dashboard → your service → **Environment** tab
2. Find `ALLOWED_ORIGINS`
3. Set its value to your Vercel URL:
   ```
   https://ai-resume-forge.vercel.app
   ```
4. Click **Save** — Render will automatically redeploy

This means only YOUR frontend can call YOUR backend. Strangers can't abuse your OpenAI API key.

## ⚠️ Free Tier Limitations

### Render Free Tier
| Limitation | Detail |
|---|---|
| **Spin-down** | Server sleeps after 15 minutes of no traffic |
| **Wake-up time** | First request after sleep takes ~30 seconds |
| **Hours** | 750 free hours/month (enough for 1 service) |
| **PDF export** | Not available (no apt-get on free tier) |

> **Spin-down fix:** When a user visits your Vercel frontend, the first request
> to the backend will be slow (~30s) if it was asleep. This is normal on the
> free tier. Upgrade to Render Starter ($7/month) to disable spin-down.

### Vercel Free Tier
| Limitation | Detail |
|---|---|
| **Bandwidth** | 100GB/month (plenty for a resume tool) |
| **Deployments** | Unlimited |
| **Custom domain** | Supported on free tier |

## 🐛 Common Deployment Problems

### "Application failed to respond" on Render
- Check the **Logs** tab in Render dashboard
- Most common cause: `OPENAI_API_KEY` environment variable not set
- Fix: Render dashboard → Environment → add `OPENAI_API_KEY`

### "CORS error" in browser console
- Your `ALLOWED_ORIGINS` on Render doesn't include your Vercel URL
- Fix: update `ALLOWED_ORIGINS` in Render environment variables

### Frontend shows "localhost" errors
- `build.js` didn't run, or `RENDER_API_URL` not set in Vercel
- Fix: Vercel dashboard → Settings → Environment Variables → add `RENDER_API_URL`
- Then: Vercel dashboard → Deployments → **Redeploy**

### PDF downloads as DOCX
- LibreOffice not available on Render free tier — this is expected
- Tell users to export to PDF from Word after downloading the DOCX
- Or upgrade Render to Starter plan ($7/month)

### First request takes 30 seconds
- Render free tier spins down after 15 minutes idle — this is normal
- Upgrade to Render Starter to remove spin-down
